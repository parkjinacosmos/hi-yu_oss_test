2. JavaScript는 동적 타이핑 타입 시스템입니다. <=> C/C++은 정적 타이핑 타입 시스템입니다.

## 변수에 대해 설명하고 let, const, var 차이점을 작성해주세요
변수(Variable)는 값이 변할 수 있거나 미지수인 데이터를 저장하고 표현하는 기호, 임시 저장 공간입니다. -> 데이터를 담는 메모리 상자
var - 기본 변수 선언으로, 중복 선언과 재할당(값 변경) 가능하며, 선언 전에도 참조할 수 있습니다.
let - 진짜 기본적인 변수 선언으로, 중복 선언 불가능(Syntax Error)이 불가능하고 재할당이 가능합니다. 블록 단위 스코프인 {}안에서 선언 시 이 블록 안에서만 참조 가능합니다.
const- 상수 선언으로, 중복 선언과 재할당 모두 불가능하며, 반드시 선언 시 값을 할당해 주어야 합니다. 블록 단위 스코프인 {} 안에서 선언 시 이 블록 안에서만 참조가 가능합니다.
변수 선언에는 기본적으로 const를 사용하며, 재할당 필요시 let 사용을 권장합니다.

## function foo() {}와 var foo = function() {} 사이에서 foo 사용의 차이에 대해 작성해주세요
- function foo() {}는 함수 선언문으로, 코드가 실행되기 전에 함수 전체가 호이스팅됩니다. (코드를 실행하기 전에, 변수나 함수의 선언 부분을 스코프(유효 범위)의 맨 위로 끌어올리는 것)
- var foo = function() {}는 함수 표현식으로, 변수 foo만 호이스팅되고, 함수 할당은 코드가 그 줄에 도달했을 때 이루어집니다. 따라서 선언하기 전에 호출하면 TypeError가 발생합니다.

## 주제1: 배열과 객체
Q. JavaScript에서 배열 = 객체인가요? 맞다면 다른 언어의 캡슐화 등과 같은 객체의 특징을 갖고 있나요?
A. JavaScript에서 배열은 객체가 맞습니다. 객체의 기본 특징이 프로퍼티와 메서드를 가집니다. (배열도 객체이기 때문에 length와 같은 프로퍼티, .push(), .pop()같은 메서드를 모두 가지고 있습니다.), 캡슐화의 기본 의미인 관련 있는 데이터와 기능을 하나로 묶는다는 관점에서는 맞지만, Java나 C++에서 말하는 캡슐화의 핵심인 데이터 숨기기(보호) 기능은 없습니다. 기본 배열의 데이터는 밖에서 누구나 볼 수 있고, 마음대로 바꿀 수 있으므로, 정확하게 캡슐화가 맞다고 말하기 어렵습니다. 따라서 배열 = 객체이지만, 다른 언어에서 캡슐화와 같은 객체의 특징을 가지고 있다고 말하기는 어렵습니다.

## 주제2: console.log와 return의 차이
Q. 함수를 만들 때 어떤 때는 console.log()로 값을 확인하고, 어떤 때는 return으로 값을 반환하던데, 이 둘은 언제 어떻게 구분해서 써야 하나요? console.log()로 화면에 나오기만 하면 이미 끝난 거 아닌가요?
A. console.log()는 단순히 개발자가 눈으로 확인하기 위해 콘솔 창에 값을 출력만 해주는 기능이고, 프로그램 내부적으로 그 값을 다시 쓸 수는 없습니다. 
return은 함수가 실행을 끝나고 그 결과값을 자신이 호출한 곳으로 반환하는 기능입니다. 반환된 값은 다른 변수에 저장하거나 다른 계산에 재사용할 수 있습니다.

## 주제3 JavaScript와 Java의 차이
Q. 이번 수업시간에 JavaScript를 배우면서, Java와는 완전히 다른 언어라는 것을 알았습니다. C나 C++은 비슷한 특징을 가지고 있었는데, 왜 JavaScript와 Java는 이름도 비슷한데 아예 다른 언어인가요? 
A. Java는 백엔드쪽에서 사용하는 언어이고, 정적 타이핑으로 관리하는 반면 JavaScript는 프론트엔드쪽에서 사용하는 언어로, 동적 타이핑으로 관리합니다. Java가 유행할 때, 라이브스크립트 대신 자바스크립트로 이름을 변경하여 출시하는 마케팅 전략을 내세웠습니다. 따라서 C에서 파생된 언어인 C++과는 아예 다른 경우입니다. 아예 언어 자체가 다릅니다!!!!
